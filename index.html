Prototypes exploratoires sur l'attestation et la défendabilité de claims numériques. Pas des outils métier — des terrains d'expérience.

⚖️ Prototype v1 · 2026-05

VERITAS Stack

A symbolic linter for numerical scientific claims.

Pipeline en six couches Aether : Joseph (génération de claims auto-attestées), Mané Thékel Pharès (attestation arithmétique), Sept Sceaux (structure 7 dimensions), VERITAS (cross-link formule ↔ raisonnement), Pentecôte (consensus 12 témoins), Tribunal (réfutation par 7 objections). Vérifie mécaniquement la cohérence et la défendabilité d'une claim numérique.

  • ~1 600 lignes Aether · ~140 fonctions
  • 300+ @example · 80+ @invariant
  • 400+ probes adverses, 0 violation
  • Benchmark 170 tests · case study 10 claims
⚠️ Prototype de recherche — ne remplace pas la peer review humaine ni l'expertise scientifique. Ne lit pas les papiers tout seul : un humain encode les métadonnées, le stack applique une grille de décision stable.

Méthode

Invariants & auditabilité

Chaque calcul critique est encadré par des invariants formels vérifiés à l'exécution. Si un invariant est violé, l'app affiche un toast d'erreur visible au lieu de produire un résultat silencieusement faux. 

// FireOps : pression à la pompe
pression_pompe(p_lance, perte, denivele) {
  // ...calcul...
  this._check(result >= 0 && result <= 16,
    'pression_pompe hors bornes');
  return result;
}
// MedTriage : invariant de triage
triage(n2, qs, ht) {
  // @invariant qSOFA ≥ 2 → triage = CRITICAL
  Engine._assert(criticalRequired === isCritical,
    'Triage CRITICAL invariant mismatch');
}
